# Политика безопасности

Не публикуйте уязвимости, реальные учётные данные, адреса прокси, токены или клиентские данные в открытых issues и pull requests.

До подключения приватного канала GitHub Security Advisories чувствительные сообщения следует направлять владельцу репозитория через закрытый контакт, указанный в профиле автора.

## Что не должно попадать в репозиторий

- реальные `PROXY_URL`, логины и пароли;
- API-ключи и токены;
- данные клиентов;
- внутренние адреса инфраструктуры;
- результаты тестов, позволяющие идентифицировать клиента или его задачу.

Примеры должны использовать только безопасные placeholders и файлы `.env.example`.
